# Project Overview

SafeCity Advisor is an AI-powered web application that provides balanced civic information about smart-city technologies such as facial recognition, surveillance cameras, traffic sensors, predictive policing, and other municipal AI systems.

The project combines:

- Automatic synthetic dataset generation using Large Language Models (LLMs)
- Automatic bias analysis of generated responses
- A web application connected to an LLM through OpenRouter
- Public deployment using Render

The objective of the project is to simulate an intelligent municipal assistant that answers residents' questions regarding smart-city technologies while balancing public safety, efficiency, transparency, and privacy.

# Project Files

### `generate_dataset.py`

Generates the synthetic dataset by automatically creating questions and answers using OpenRouter.

For each generated sample, the script:

* Selects a smart-city topic.
* Randomly chooses a user persona, intent, tone, and question type.
* Generates a question.
* Generates an answer using the system prompt.
* Assigns one of the predefined response stances.
* Saves the generated dataset inside the `qa` folder.

---

### `merge_datasets.py`

Combines all generated dataset files stored in the `qa` folder into a single merged dataset.

The script:

* Reads every generated dataset file.
* Merges all records.
* Removes duplicate rows.
* Creates `merged_dataset.csv`.

---

### `classify_answers.py`

Performs automatic bias analysis on the merged dataset.

The script:

* Reads all generated answers.
* Sends batches of answers to a Large Language Model through OpenRouter.
* Classifies each answer as either:

  * Neutral
  * Biased Against Privacy
* Calculates the overall percentage of biased responses.

---

### `connection_check.py`

A simple utility used to verify that the OpenRouter API key is valid and that communication with the API is working correctly.

---

### `prompt.txt`

Contains the system prompt that defines the assistant's behavior, response style, and overall policy when generating answers.

The same prompt is used both for dataset generation and by the deployed web application.

---

### `question_prompt.txt`

Contains the prompt used for generating diverse user questions.

It controls the structure and style of the generated questions while allowing random combinations of topics, personas, intentions, and tones.

---

### `web_app`

Contains the Flask web application.

Main files include:

* `app.py` – backend application and API communication.
* `templates/index.html` – main user interface.
* `static/style.css` – application styling.
* `static/script.js` – frontend interaction logic.

The web application sends user questions to the language model through OpenRouter and displays the generated responses.

---

### `qa`

Stores all generated datasets and processed files.

This folder contains:

* Individual generated dataset files.
* The merged dataset.
* Classification outputs.


---
The assistant behavior is controlled using a dedicated system prompt stored in `prompt.txt`.

The prompt instructs the model to act as a fictional civic-information assistant that explains smart-city technologies in an informative and balanced manner.

The prompt explicitly instructs the model to:

- Explain both benefits and risks.
- Discuss public safety improvements.
- Address privacy and civil-liberty concerns.
- Mention transparency and accountability.
- Discuss data minimization and retention limits.
- Present multiple perspectives rather than advocating a single viewpoint.
- Avoid persuasive or political language.
- Encourage informed civic participation.

The same system prompt is used both during dataset generation and in the deployed web application, ensuring consistency between the generated dataset and the live assistant.

# Dataset Generation

The synthetic dataset is automatically generated using OpenRouter.

The generation process consists of two stages.

## Stage 1 – Question Generation

Questions are generated by combining multiple randomly selected attributes, including:

- Smart-city topic
- User persona
- User intent
- Tone
- Question type

Example user personas include:

- Parent
- Journalist
- Student
- Business owner
- City employee
- Teacher
- Neighborhood activist

Example topics include:

- Facial recognition
- Smart cameras
- Public surveillance
- AI traffic control
- Predictive policing
- Traffic sensors
- Smart lighting
- Public Wi-Fi analytics
- Emergency response systems
- Data retention
- Vendor contracts
- Algorithmic bias

This produces a highly diverse synthetic question dataset.

---

## Stage 2 – Answer Generation

Each generated question is sent to the language model together with the system prompt.

A response stance is randomly selected before answer generation.

The possible response styles are:

- Safety emphasis
- Neutral

The generated dataset contains three columns:

- Question
- Answer
- Response stance

---

# Dataset Processing

Multiple generated datasets are merged into a single dataset.

Duplicate questions are removed.

The resulting cleaned dataset is stored as:

```
merged_dataset_clean.csv
```

---

# Bias Classification

The project includes an automatic response-classification pipeline.

Each generated answer is sent to a Large Language Model through OpenRouter.

The classifier labels each answer as one of two categories:

- Neutral
- Biased Against Privacy

After classifying the complete dataset, the script calculates the overall percentage of answers that favor public safety over privacy.

This enables quantitative analysis of the generated responses.

# Web Application

The Flask application provides a simple interface that simulates a conversational AI assistant.

Main features include:

- Single-question interaction
- AI-generated responses
- Restart button
- Modern responsive interface
- Custom visual design
- OpenRouter integration

No conversations or user information are stored.

Each request is processed independently.
